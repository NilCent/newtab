# newtab
用于提升学习效率的 newtab chrome 扩展

## 为什么开发
在近些年, 我使用过许多提升效率的工具和方法, flashcard, 看板, okr, 番茄钟等.我在使用过程中, 遇到一个对我而言十分致命的痛点 -- 这些工具没有办法 **强制** 我使用它们, 无非就是发几个提醒.

我的解决方法是把这些工具放到 newtab 里, 因为我每天访问最多的页面就是 chrome 的 newtab, 一般都会从 newtab 的搜索栏或者历史记录进入其他页面.
目前的逻辑是如果 **提醒未解决, 就禁止使用搜索栏和历史记录**.

现在开始准备 v0.4 版本的开发, 在 v0.4 版本将聚焦于开发 flashcard 组件, 想先和你沟通以下这些功能是否可以实现

flashcard 主要是将文本以卡片的形式展示, 分为只有一面的卡片, 和有正反面的卡片

- 现有的 quote 组件就可以被抽象为只有一面的卡片, 当 flashcard 组件完成后, 预计将现有的 quote 组件替换

- 拥有正反面的卡片一般正面是问题, 反面是答案, 比如生词本之类的

我想扩展 markdown 的语法后, 实现扩展的 markdown 来编写 flashcard, 这样我就可以使用任意编辑器修改 flashcard, 也能很方便的存储在 github 中. 扩展的语法规则如下

- 单行正反面卡片, 在一行内 ">>>" 左侧是正面, 右侧是反面, 比如 "immune >>> 免疫"

- 多行正反面卡片, 使用一对 ":::" 包裹的内容为反面, 第一个 ::: 前的内容为正面

- 只有一面卡片, 每张卡片使用 "***" 分割

第一个问题是这个扩展语法可以实现吗?

完成上述扩展语法后, flashcard 会写在本地的文件中, 想通过 github 来管理.

第二个问题是可以在 chrome 扩展里操作本地文件吗? 可以控制本地仓库吗?
